# UniType Engine

**UniType** — это высокопроизводительная, header-only C++17 библиотека для N-мерных математических и физических симуляций. 

Она построена на принципах **Data-Oriented Design (DOD)** и **Structure of Arrays (SoA)**, что позволяет достичь экстремальной производительности за счет идеального использования кэша процессора и автоматической SIMD-векторизации (Intel TBB).

Библиотека универсальна: она подходит для расчета орбитальной механики, диффузии тепла, динамики N-тел, обучения нейросетей и симуляций методом Монте-Карло.

## 🚀 Главные возможности

* **Универсальная размерность:** Шаблонная архитектура `template <typename T, size_t N>` позволяет работать с любыми размерностями (от скаляров до многомерных тензоров) и любыми типами данных (double, std::complex, кастомные классы).
* **Zero-cost Абстракции:** Пользователь пишет красивый ООП-код (через "Линзы" / Proxies), который на этапе компиляции разворачивается в прямые обращения к плоским массивам в памяти.
* **Многопоточность из коробки:** Операции над массивами размером больше 1000 элементов автоматически распараллеливаются с помощью `std::execution::par_unseq`.
* **Zero-copy Аналитика:** Извлечение истории (временных рядов) для конкретного объекта происходит без копирования памяти, с использованием математических смещений (strides).

## 🏗 Архитектура (DOD)

Вместо классического ООП, где каждый объект хранит свои данные (Массив Структур - AoS), UniType использует единые плоские массивы для всей системы (Структура Массивов - SoA).

Библиотека состоит из 3 независимых модулей:
1. `UnitType.h` — Математическое ядро (N-мерный вектор со статистикой и векторной алгеброй).
2. `SystemState.h` — "Слепок" системы в текущий момент времени. Хранит "координаты", "скорости" и "ускорения" (величины и их производные 1 и 2 порядка) N объектов в непрерывной памяти.
3. `History.h` — Многоканальный архив. Эффективно сохраняет историю (эфемериды в случае орбит) и предоставляет доступ к аналитике по любому объекту (аналог колонок в `pandas` или `NumPy`).

## 💻 Быстрый старт (Quick Start)

Пример симуляции движения N-мерных объектов и сохранения их истории:

```cpp
#include <iostream>
#include "unitype/UnitType.h"
#include "unitype/SystemState.h"
#include "unitype/History.h"

using namespace unitype;

int main() {
    const size_t NUM_OBJECTS = 100'000;
    const size_t NUM_STEPS = 1000;

    // 1. Создаем систему и архив
    SystemState<double, 3> system(NUM_OBJECTS);
    HistoryContainer<double, 3> archive(NUM_OBJECTS, NUM_STEPS);

    // 2. Инициализируем начальные скорости через ООП-фасад (Proxy)
    for (size_t i = 0; i < system.size(); ++i) {
        system[i].velocity() = {1.0, 2.0, 3.0};
    }

    // 3. Главный цикл интегратора
    double dt = 0.5;
    double current_time = 0.0;

    for (size_t step = 0; step < NUM_STEPS; ++step) {
        // Шаг физики (например, метод Эйлера)
        for (size_t i = 0; i < system.size(); ++i) {
            system[i].coords() = system[i].coords() + system[i].velocity() * dt;
        }
        
        // Сверхбыстрое сохранение всего "кадра" в архив
        archive.push_step(current_time, system.get_coords());
        current_time += dt;
    }

    // 4. Аналитика временного ряда для объекта №42 (Zero-copy)
    auto track42 = archive.get_track(42);
    std::cout << "Максимальное отклонение: " << track42.max_abs() << "\n";

    return 0;
}

🛠 Требования и сборка

Библиотека является header-only. Вам не нужно её собирать, достаточно просто подключить заголовочные файлы в ваш проект.

Требования:

    Компилятор с поддержкой C++17 или выше.

    Установленная библиотека Intel TBB (Threading Building Blocks) для параллельных алгоритмов.

Пример компиляции через GCC (MinGW на Windows):

g++ -std=c++17 -O3 -I./src my_program.cpp -o my_program.exe -L C:/msys64/ucrt64/lib -ltbb12 "-Wl,--stack,268435456"
